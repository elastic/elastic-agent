# Buildkite GCS Artifacts - Terraform Module

This Terraform module automates the setup of Google Cloud Storage for Buildkite artifacts with OIDC authentication via Workload Identity Federation.

## What This Creates

- **GCS Bucket** (`bk-elastic-agent`): Private bucket for storing artifacts
- **Workload Identity Pool**: OIDC integration between Buildkite and GCP
- **OIDC Provider**: Configured with Buildkite as the issuer
- **Service Account**: For Buildkite agents to access GCS
- **IAM Bindings**: Permissions for service account and Elastic users
- **Lifecycle Policies**: Automatic cleanup of old artifacts

## Prerequisites

1. **GCP Project**: You need a GCP project with billing enabled
2. **Terraform**: Install [Terraform](https://www.terraform.io/downloads) >= 1.0
3. **GCP Credentials**: Authenticate with `gcloud auth application-default login`
4. **Permissions**: You need Owner or Editor role on the GCP project

## Quick Start

### 1. Configure Variables

```bash
cd .buildkite/terraform/gcs-artifacts
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your values
```

### 2. Initialize Terraform

```bash
terraform init
```

### 3. Review the Plan

```bash
terraform plan
```

Expected resources:
- 1 GCS bucket
- 1 Workload Identity Pool
- 1 OIDC Provider
- 1 Service Account
- 2-3 IAM bindings

### 4. Apply the Configuration

```bash
terraform apply
```

Type `yes` to confirm.

### 5. Capture Outputs

```bash
# Get the environment hook configuration
terraform output -raw buildkite_environment_hook > ../../../hooks/environment
chmod +x ../../../hooks/environment

# Get the plugin configuration (for reference)
terraform output -raw buildkite_plugin_config > plugin-config.txt
cat plugin-config.txt
```

### 6. Update Pipelines

Add the plugin configuration from `plugin-config.txt` to your pipeline steps.

## Outputs

| Output | Description |
|--------|-------------|
| `bucket_name` | GCS bucket name |
| `bucket_url` | Full bucket URL |
| `service_account_email` | Service account email for the plugin |
| `workload_identity_provider` | Full provider resource name |
| `plugin_audience` | Audience value for the Buildkite plugin |
| `buildkite_environment_hook` | Ready-to-use environment hook script |
| `buildkite_plugin_config` | Ready-to-use plugin configuration |

## Configuration Options

### Bucket Location

Choose a location close to your Buildkite agents:

```hcl
bucket_location = "us-central1"  # Single region (cheaper)
# or
bucket_location = "US"  # Multi-region (higher availability)
```

### Pipeline Restrictions

Allow only specific pipelines:

```hcl
allowed_pipeline_slugs = ["elastic-agent", "elastic-agent-package"]
```

Or allow all pipelines in the organization:

```hcl
allowed_pipeline_slugs = []  # Default
```

### Retention Policies

Adjust retention based on your needs:

```hcl
artifact_retention_days      = 30  # Main branch artifacts
test_artifact_retention_days = 7   # PR/test artifacts
```

### User Access

Grant read access to a Google Group:

```hcl
elastic_users_group = "buildkite-users@elastic.co"
```

Or use a domain:

```hcl
# In main.tf, change the member to:
member = "domain:elastic.co"
```

## Testing

After applying the Terraform configuration, test the setup:

```bash
# 1. Test OIDC token generation (from a Buildkite agent)
buildkite-agent oidc request-token \
  --audience "$(terraform output -raw plugin_audience)" \
  --format gcp

# 2. Create a test pipeline
cat > test-artifacts.yml <<EOF
steps:
  - label: ":arrow_up: Upload test"
    command: |
      echo "test content" > test.txt
      buildkite-agent artifact upload "test.txt"
    plugins:
      - gcp-workload-identity-federation#v1.5.0:
          audience: "$(terraform output -raw plugin_audience)"
          service-account: "$(terraform output -raw service_account_email)"

  - label: ":arrow_down: Download test"
    depends_on: "upload-test"
    command: |
      buildkite-agent artifact download "test.txt" .
      cat test.txt
    plugins:
      - gcp-workload-identity-federation#v1.5.0:
          audience: "$(terraform output -raw plugin_audience)"
          service-account: "$(terraform output -raw service_account_email)"
EOF

# 3. Trigger the pipeline
buildkite-agent pipeline upload test-artifacts.yml
```

## Troubleshooting

### "Bucket already exists"

If the bucket name is taken:

```hcl
# In terraform.tfvars, use a unique name
bucket_name = "bk-elastic-agent-UNIQUE_SUFFIX"
```

### "Permission denied"

Ensure you have the required GCP permissions:

```bash
# Check your current authentication
gcloud auth list

# Re-authenticate if needed
gcloud auth application-default login

# Verify project
gcloud config get-value project
```

### "Invalid audience"

The audience must exactly match the Workload Identity Provider name:

```bash
# Get the correct audience
terraform output plugin_audience

# Use it in your pipeline (without https:// prefix)
```

### "Attribute condition failed"

Verify the organization slug matches:

```bash
# Check the Buildkite organization slug
# It should match the value in terraform.tfvars
```

## Cleanup

To destroy all resources:

```bash
terraform destroy
```

**Warning:** This will delete the bucket and all artifacts!

## Manual Verification

After Terraform apply, verify the setup manually:

```bash
# 1. Check bucket exists and is private
gsutil ls gs://bk-elastic-agent
gsutil iam get gs://bk-elastic-agent

# 2. Check workload identity pool
gcloud iam workload-identity-pools describe buildkite-artifacts-pool \
  --location=global \
  --project=$(terraform output -raw project_id)

# 3. Check OIDC provider
gcloud iam workload-identity-pools providers describe buildkite-oidc-provider \
  --workload-identity-pool=buildkite-artifacts-pool \
  --location=global \
  --project=$(terraform output -raw project_id)

# 4. Check service account
gcloud iam service-accounts describe $(terraform output -raw service_account_email)

# 5. Check IAM bindings
gcloud iam service-accounts get-iam-policy $(terraform output -raw service_account_email)
```
